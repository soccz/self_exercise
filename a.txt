  ## 네가 직접 해야 할 것 (체크리스트)

  1. Supabase SQL 1회 실행

  - Supabase SQL Editor에서 supabase/telegram_reminder_patch.sql 실행
  - 리마인더 설정/중복방지 컬럼이 생겨야 /remind와 크론이 정상 동작

  2. Vercel 환경변수 확인/등록

  - SUPABASE_URL
  - SUPABASE_SERVICE_ROLE_KEY
  - NEXT_PUBLIC_SUPABASE_URL
  - NEXT_PUBLIC_SUPABASE_ANON_KEY
  - TELEGRAM_BOT_TOKEN
  - (권장) TELEGRAM_WEBHOOK_SECRET
  - APP_URL (버셀 도메인)
  - (권장) APP_SECRET (앱 잠금 + 크론 인증에 필요)

  3. Telegram webhook 확인

  - BotFather 토큰이 제대로 들어갔는지
  - webhook이 https://<vercel-domain>/api/telegram으로 걸려있는지
  - TELEGRAM_WEBHOOK_SECRET 쓴다면, secret_token도 일치하는지

  4. GitHub Actions Secrets 등록 (리마인더용)

  - APP_SECRET (Vercel과 동일)
  - CRON_ENDPOINT = https://<vercel-domain>/api/cron/remind
  - Actions 탭에서 telegram-reminder 워크플로우 수동 Run 1번 해서 200으로 떨어지
    는지 확인

  ## 동작 확인(추천 순서, 3분)

  1. 텔레그램에서 /help 확인
  2. 텔레그램에서 /remind on → /remind test
  3. 웹에서 운동 하나 저장 → 홈 자산/분석에 반영되는지
  4. 웹에서 오프라인(와이파이 끊기) 상태로 저장 1번 → 온라인 복귀 후 자동 반영되
     는지(프로필에 “대기 N건” 표시 확인)
